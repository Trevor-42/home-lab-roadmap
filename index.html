Pro Tips
1
Label every cable end — both ends, with masking tape and a marker or a label maker. You'll thank yourself 6 months later when you need to swap something.
2
Use short patch cables — 1ft and 2ft cables in different colors for each device. Keeps the rack tight and makes tracing connections instant.
3
Keep 6"+ clearance behind and above the NAS for airflow. The DS423+ pulls air through the drives and exhausts out the back — blocking it will raise temperatures and shorten drive life.
4
Run the Mac Mini headless — after initial setup, SSH and VNC mean you never need a monitor plugged into it. Saves a cable and keeps the install tidy.
5
Set static IPs on your router for the Mini and NAS — assign them fixed local IPs (e.g. 192.168.1.10 and 192.168.1.11). Everything stays consistent and you never have to hunt for device addresses.

Project 08 — Package Integrity
Certified Package — Chain of Custody & Theft Claim System
A tamper-evidence and chain-of-custody app for shipping Pokémon cards and collectibles. Photograph and timestamp contents before sealing, track the package through transit, detect tampering or missing contents on arrival, and auto-generate pre-filled claim paperwork for USPS, FedEx, UPS, and others. Strongest standalone product — strong SaaS potential beyond personal use.
Advanced
The Real Problem — Documented Cases
A former USPS sorting clerk in NJ pleaded guilty to stealing $108,692 in Pokémon and sports cards across 10 packages. Mail theft is a federal crime (up to 5 years prison) but victims routinely lose claims because they lack timestamped proof of what was inside before sealing. USPS rejects 30–40% of all insurance claims due to insufficient documentation — this app solves that problem directly.

Project 10 — Video Editing
AI-Assisted Video Editing Pipeline
Local AI video editing toolkit — auto-generate cuts, captions, thumbnails, and social-ready clips from raw footage. Process everything on your Mac's GPU with no cloud subscriptions. Powers content for Project 11 (Social Media) and Etsy product promo videos.
Moderate
Why Local Video Editing Matters
Cloud video tools (CapCut Pro, Descript, Opus Clip) cost $15–$30/month each and upload your content to third-party servers. A local pipeline keeps footage private, runs offline, and costs $0/month after setup. Apple Silicon's hardware video encoders make this practical — M4/M5 Pro can export 4K H.265 faster than real-time.
Architecture
IngestDrop raw footage into watch folder on NAS or local SSD
↓
TranscribeWhisper (local) — full transcript with word-level timestamps
↓
AI EditLLM selects best clips, removes silence/filler, suggests cuts
↓
RenderFFmpeg — auto-caption, crop to 9:16/1:1, export per platform
↓
OutputPlatform-ready clips: TikTok, Reels, Shorts, full YouTube
Build Phases
1
Whisper transcription pipeline
Install whisper.cpp (Apple Silicon optimized). Batch-process video files into timestamped transcripts. This is the foundation — every downstream edit decision uses the transcript.
2
Smart clip extraction
Feed transcript to local LLM with prompt: "Find the 5 most engaging 30–60 second segments." LLM returns timestamp ranges. FFmpeg extracts clips automatically. Silence detection removes dead air.
3
Auto-captioning & formatting
Burn word-by-word animated captions (SRT → ASS styling). Auto-crop to 9:16 for TikTok/Reels, 1:1 for Instagram feed, 16:9 for YouTube. One source video → multiple platform exports.
4
Thumbnail & title generation
Extract highest-energy frame per clip. LLM generates clickable titles and descriptions per platform. Optional: FLUX generates custom thumbnail backgrounds from text prompts.
5
Watch folder automation
Drop a video file into a folder → full pipeline runs automatically. Output: captioned clips + thumbnails + metadata JSON ready for Project 11 (Social Media) to post.

📘 Step-by-Step Guides
Guide 1: Install Ollama on macOS
Time: ~2 minutes · Difficulty: Easy · Prerequisite: None
1
Download Ollama
Go to ollama.com/download and click "Download for macOS". This downloads a .zip file (~60MB).
Or use terminal: brew install ollama
2
Install the App
Unzip and drag Ollama.app to your Applications folder. Double-click to launch. You'll see a llama icon appear in your menu bar — that means the server is running.
3
Verify Installation
Open Terminal and run:
$ ollama --version
ollama version 0.6.x

$ ollama list
# Empty table if first install — that's normal
✓
Done!
Ollama runs as a background service. It starts automatically on boot. API available at http://localhost:11434
Guide 2: Download & Run Your First Model
Time: 5–30 min (depends on model size + internet) · Prerequisite: Ollama installed
1
Pull a Model
Open Terminal and pick a model. Start small to test, then go bigger:
# Small & fast (2GB download, good for testing)
$ ollama pull llama3.2:3b

# Medium — great daily driver (20GB download)
$ ollama pull qwen3:32b

# Large — near GPT-4 quality (40GB download)
$ ollama pull llama3.3
You'll see a progress bar. Models download once and are cached at ~/.ollama/models/
2
Chat with the Model
Run starts an interactive chat session:
$ ollama run llama3.2:3b
>>> What is the capital of France?
The capital of France is Paris...

>>> Write a Python function to sort a list
def sort_list(items): ...

# Type /bye to exit the chat
>>> /bye
3
Check What's Installed
$ ollama list
NAME             SIZE   MODIFIED
llama3.2:3b     2.0 GB  5 minutes ago
llama3.3:latest  39 GB  2 minutes ago

# Delete a model you don't need
$ ollama rm llama3.2:3b
✓
Tips
• ollama run auto-downloads if model not found — no need to pull first
• Only one model loads at a time (Ollama swaps automatically)
• Models stay in memory for ~5 min after last use, then unload
Guide 3: Use the API in Your Code
Time: ~5 minutes · Prerequisite: Ollama running with a model pulled
1
Test with curl (OpenAI-compatible)
Ollama exposes an OpenAI-compatible API. Any code that works with OpenAI works with Ollama — just change the base URL:
$ curl http://localhost:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "llama3.2:3b",
    "messages": [
      {"role": "user", "content": "Hello!"}
    ]
  }'
2
Python (with OpenAI SDK)
$ pip install openai

from openai import OpenAI

client = OpenAI(
  base_url="http://localhost:11434/v1",
  api_key="ollama"  # required but unused
)

response = client.chat.completions.create(
  model="llama3.2:3b",
  messages=[{"role": "user", "content": "Hello!"}]
)
print(response.choices[0].message.content)
3
JavaScript / Node.js
$ npm install ollama

import { Ollama } from 'ollama'

const ollama = new Ollama()

const response = await ollama.chat({
  model: 'llama3.2:3b',
  messages: [{ role: 'user', content: 'Hello!' }]
})
console.log(response.message.content)
4
Ollama Native API
Ollama also has its own API (non-OpenAI) with extra features like streaming, model management, and embeddings:
# Generate (non-chat, single prompt)
POST http://localhost:11434/api/generate

# Chat (multi-turn conversation)
POST http://localhost:11434/api/chat

# Embeddings (for RAG/search)
POST http://localhost:11434/api/embed

# List models / pull / delete
GET  http://localhost:11434/api/tags
POST http://localhost:11434/api/pull
DELETE http://localhost:11434/api/delete
Full API docs: github.com/ollama/ollama/blob/main/docs/api.md

Guide 5: Set Up Open WebUI (ChatGPT-style Interface)
Time: ~5 minutes · Prerequisite: Ollama running + Docker installed
1
Install Docker (if not already)
Download from docker.com or:
$ brew install --cask docker
2
Launch Open WebUI
One command. It auto-connects to your local Ollama:
$ docker run -d \
  -p 3000:8080 \
  --add-host=host.docker.internal:host-gateway \
  -v open-webui:/app/backend/data \
  --name open-webui \
  --restart always \
  ghcr.io/open-webui/open-webui:main
3
Open in Browser
Go to http://localhost:3000. Create an admin account on first visit. Select any Ollama model from the dropdown. Chat away — conversation history is saved automatically.
4
Access from Anywhere (via Tailscale)
If Tailscale is set up on your Mac Mini, access Open WebUI from any device on your tailnet:
# From your phone/laptop on tailnet:
http://mac-mini:3000
# or use Tailscale IP:
http://100.x.x.x:3000
Full ChatGPT-like experience, running on your hardware, accessible from the field. Zero cloud costs.
Guide 6: Set Up Embeddings for RAG / Semantic Search
Time: ~10 minutes · Prerequisite: Ollama running + Python
1
Pull an Embedding Model
$ ollama pull nomic-embed-text
# ~275MB download, runs instantly
2
Generate Embeddings
import httpx

r = httpx.post("http://localhost:11434/api/embed", json={
  "model": "nomic-embed-text",
  "input": [
    "Charizard VMAX is worth $200",
    "Fire-type Pokémon cards are popular",
    "The weather is sunny today"
  ]
})
embeddings = r.json()["embeddings"]
# Returns list of 768-dim vectors
print(len(embeddings[0])) # 768

Guide 7: Model Management & Tips
Essential commands and tricks for managing your local model library
Essential Commands
# List installed models
ollama list
# See running models
ollama ps
# Model details (size, params, quantization)
ollama show llama3.3
# Delete a model
ollama rm llama3.2:3b
# Update to latest version
ollama pull llama3.3
# Copy/rename a model
ollama cp llama3.3 my-llama
Size & Performance Tips
VRAM rule: model needs ~60% of param count in GB at Q4 quantization. 70B → ~40GB.

Speed estimate (M4 Pro):
• 3B → ~80 tok/s
• 7B → ~40 tok/s
• 14B → ~25 tok/s
• 32B → ~12 tok/s
• 70B → ~5 tok/s

Quantization: default Q4_K_M is best balance. Q8 = more accurate but 2x size. Q2 = tiny but noticeable quality loss.

Storage: models at ~/.ollama/models/. Move to external drive with symlink if SSD fills up.
💡 Pro Tip: Custom System Prompts
Create a Modelfile to customize any model with a permanent system prompt:
# Create a file called "Modelfile"
FROM llama3.3
SYSTEM "You are a Rhombus security camera expert. Help installers with camera placement, network config, and troubleshooting."
PARAMETER temperature 0.3

# Build and run your custom model
$ ollama create rhombus-helper -f Modelfile
$ ollama run rhombus-helper
